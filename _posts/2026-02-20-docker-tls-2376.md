# Docker 开启 TLS 加密远程访问（2376 端口）

默认情况下 Docker 仅监听本地 Unix socket，如需从远程机器管理 Docker（如 CI/CD 系统、Portainer 等），需要开放 TCP 端口。**直接暴露 2375 明文端口存在严重安全风险**，应使用 2376 端口配合 TLS 双向认证。

## 整体流程

1. 生成 CA 证书（根证书）
2. 生成服务端证书并用 CA 签名
3. 生成客户端证书并用 CA 签名
4. 配置 Docker 使用 TLS
5. 客户端连接验证

---

## 一、证书生成脚本

将以下脚本保存为 `docker_cert.sh`，**修改 `ip` 为服务器实际 IP，`password` 替换为强密码**：

```bash
#!/bin/sh

ip=YOUR_SERVER_IP       # 替换为实际的服务器 IP
password=YOUR_PASSWORD  # 替换为证书密码
dir=/etc/docker/certs   # 证书存放路径

# 创建目录（已存在则清空重建）
if [ ! -d "$dir" ]; then
  mkdir -p "$dir"
else
  rm -rf "$dir"
  mkdir -p "$dir"
fi

cd "$dir"

# 1. 创建 CA 私钥
openssl genrsa -aes256 -passout pass:$password -out ca-key.pem 4096

# 2. 创建 CA 证书
openssl req -new -x509 -days 3650 -key ca-key.pem \
  -passin pass:$password -sha256 -out ca.pem \
  -subj "/C=CN/ST=Beijing/L=Beijing/O=MyOrg/CN=$ip"

# 3. 创建服务端私钥和签名请求
openssl genrsa -out server-key.pem 4096
openssl req -subj "/CN=$ip" -sha256 -new -key server-key.pem -out server.csr

# 4. 配置服务端证书的 SAN
echo "subjectAltName = IP:$ip,IP:127.0.0.1" > extfile.cnf
echo "extendedKeyUsage = serverAuth" >> extfile.cnf

# 5. 签发服务端证书
openssl x509 -req -days 3650 -sha256 -in server.csr \
  -CA ca.pem -CAkey ca-key.pem -passin "pass:$password" \
  -CAcreateserial -out server-cert.pem -extfile extfile.cnf

# 6. 创建客户端私钥和签名请求
openssl genrsa -out key.pem 4096
openssl req -subj '/CN=client' -new -key key.pem -out client.csr

# 7. 签发客户端证书
echo "extendedKeyUsage = clientAuth" > extfile-client.cnf
openssl x509 -req -days 3650 -sha256 -in client.csr \
  -CA ca.pem -CAkey ca-key.pem -passin "pass:$password" \
  -CAcreateserial -out cert.pem -extfile extfile-client.cnf

# 8. 清理中间文件
rm -f client.csr server.csr extfile.cnf extfile-client.cnf

# 9. 设置权限（私钥只读）
chmod 0400 ca-key.pem key.pem server-key.pem
chmod 0444 ca.pem server-cert.pem cert.pem

echo "证书生成完成，路径：$dir"
ls -la "$dir"
```

执行脚本：

```bash
chmod +x docker_cert.sh
./docker_cert.sh
```

生成的证书文件：

| 文件 | 说明 | 部署位置 |
|------|------|----------|
| `ca.pem` | CA 根证书 | 服务端和客户端都需要 |
| `server-cert.pem` | 服务端证书 | 服务器 |
| `server-key.pem` | 服务端私钥 | 服务器 |
| `cert.pem` | 客户端证书 | 客户端 |
| `key.pem` | 客户端私钥 | 客户端 |

## 二、配置 Docker 使用 TLS

### 方式一：修改 docker.service（推荐）

```bash
sudo vim /usr/lib/systemd/system/docker.service
```

找到 `ExecStart` 行，替换为：

```ini
ExecStart=/usr/bin/dockerd \
  -H fd:// \
  --containerd=/run/containerd/containerd.sock \
  --tlsverify \
  --tlscacert=/etc/docker/certs/ca.pem \
  --tlscert=/etc/docker/certs/server-cert.pem \
  --tlskey=/etc/docker/certs/server-key.pem \
  -H tcp://0.0.0.0:2376 \
  -H unix:///var/run/docker.sock
```

重载并重启：

```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 方式二：通过 daemon.json 配置

```bash
sudo vim /etc/docker/daemon.json
```

```json
{
  "tls": true,
  "tlscacert": "/etc/docker/certs/ca.pem",
  "tlscert": "/etc/docker/certs/server-cert.pem",
  "tlskey": "/etc/docker/certs/server-key.pem",
  "tlsverify": true,
  "hosts": ["unix:///var/run/docker.sock", "tcp://0.0.0.0:2376"]
}
```

重启 Docker：

```bash
sudo systemctl restart docker
```

验证 2376 端口已监听：

```bash
netstat -lnpt | grep 2376
```

## 三、客户端连接

将 `ca.pem`、`cert.pem`、`key.pem` 三个文件复制到客户端机器，然后：

```bash
# 测试连接
docker --tlsverify \
  --tlscacert=/path/to/ca.pem \
  --tlscert=/path/to/cert.pem \
  --tlskey=/path/to/key.pem \
  -H=<SERVER_IP>:2376 \
  version
```

将证书放到 `~/.docker/` 目录并设置环境变量，可以简化每次的参数输入：

```bash
mkdir -p ~/.docker
cp ca.pem cert.pem key.pem ~/.docker/

export DOCKER_HOST=tcp://<SERVER_IP>:2376
export DOCKER_TLS_VERIFY=1
export DOCKER_CERT_PATH=~/.docker

# 此后可以直接使用 docker 命令
docker ps
```

## 安全建议

- 证书有效期建议设置 `3650` 天（10年），避免频繁轮换，生产敏感环境可设置更短
- `ca-key.pem` 是最敏感的文件，签发完证书后可离线保存
- 防火墙应限制 2376 端口的访问来源，仅允许可信 IP
- 不要将私钥文件（`*-key.pem`）提交到 Git 仓库
